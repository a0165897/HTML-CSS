以下，是一些拥有特殊语义的格式标签。你最好按照语义使用他们，而不是格式，因为你有层叠样式表。

dfn只用来标记术语。例如：鲁棒性

code只用来标记代码。例如：int main(){printf("I think it's OK.");}

samp只用来标示会引起误解的字符，不常用。例如：字符序列 ae 可能会被转换为 æ 连字字符。

kbd只用来标示键盘输入的字符。例如：ROCK AND ROLL

cite只用来定义引用。例如：Only when it's truly dark, will you see stars.

var用来定义变量。count，常与code和pre连用来输入代码。 

int main(){
    int count;
    printf("YOOO");
}

还有一些格式标签。

mark用来高亮。例如：my english name is Mark in high school.

pre定义预格式化的文本。被包围在 pre 元素中的文本通常会保留空格和换行符。而文本也会呈现为等宽字体。见上文的代码。

progress是进度条，有二态，很有意思。（H5,IE9以前不支持）
一态：95%
二态：不标识具体值（FIREFOX会鸡贼地直接标示成100%）

time标签不会改变格式，但是浏览器和搜索引擎会识别，也许会有好事发生。
